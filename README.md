# 《台灣三國誌：萬人版》最終開發與設計全書 (Ultimate Master GDD & DDD)

## 0. 專案背景與開發導引 (Handover Brief)
*   **專案名稱**：台灣三國誌 (Chronicles of Formosa: Online SLG)
*   **核心目標**：建立一個支持 **10,000 人同服** 的大型多人在線歷史策略遊戲。
*   **開發環境**：Go 1.26.0 / Ebiten (2D) / KCP (Reliable UDP) / Protobuf (v3)。
*   **致開發者 (Gemini Pro 3.1)**：請遵循「**協定先行 (Protocol-First)**」原則，所有功能模組必須先定義於 `.proto` 檔案，嚴禁客戶端授權關鍵邏輯（戰鬥計算、資源增減）。

## 壹、 歷史世界觀：三國鼎立與族群博弈

*   **範圍**：台灣本島及周邊 80 海浬（含澎湖、黑水溝）。
*   **黑水溝系統**：具備「湍流值」。颱風季時航道封鎖，影響清軍援軍與海上貿易。
*   **瘴氣系統**：北部與山區格位具衰減傷害，需透過藥理研發或原民導遊抵銷。

## 貳、 萬人核心玩法：【庄頭共享經營】 (Joint Development)

為了解決 10,000 名玩家的空間擠壓與單打獨鬥的單調性，引入以下創新：

### 1. 庄頭 (Village) 層級化
*   **共享主體**：大地圖格位不再是單人擁有，一個「庄頭」可容納 100 名玩家。
*   **職位分工**：
    *   **庄長**：負責內政與對外外交。
    *   **墾首**：負責帶隊向瘴氣區擴張土地。
    *   **武師/勇士**：負責訓練民兵與防禦出草。
    *   **商賈**：負責管理糧食與銀兩貿易。

### 2. 天災與救災小遊戲：【牛車糧道守衛】
*   **觸發**：伺服器同步觸發地震或颱風。
*   **小遊戲**：庄長發起，玩家提供「精力值」支援。
*   **玩法**：在限時內繪製「牛車運輸路線」，避開坍方與亂民。成功後全庄獲得「重建紅利」（產值加倍）。

### 3. 族群緊張儀 (Tension Meter)
*   可視化 UI 引信。當庄內族群失衡時，引信縮短。
*   **分類械鬥**：引信燒完觸發內部戰鬥，建築物損毀。玩家需透過「辦桌、祭祀、聯姻」來延長引信。

## 參、 專案技術架構 (DDD - Technical Standards)

### 1. 協定先行原則 (Source of Truth)
*   **唯一事實檔案**：`proto/message.proto`。
*   **流程**：`Define Proto` -> `Run gen_proto.sh` -> `Implement Server Handler` -> `Update Client Listen`.
*   **禁止事項**：嚴禁手動修改 `resource/message.pb.go`。

### 2. 伺服器技術規範
*   **通訊協定**：`xtaci/kcp-go`。
*   **分區管理 (AOI)**：將萬人玩家劃分為蜂巢式節點（基隆、台中、台南、台東、澎湖），僅同步視野內格位變動。
*   **情報感測**：關鍵字截獲系統。當私聊或庄頭頻道出現「進攻」、「夜襲」等字眼，敵對陣營 NPC 說書人會生成「模糊情報」。

### 3. 前端 UI 標準 (EbitenUI Framework)
*   **i18n 多語系**：所有文字透過 `GetText("ID")` 調用。
*   **組件清單**：
    *   `GuiKeyboard`: 全局虛擬鍵盤。
    *   `GlobalToastManager`: 網路與系統提示。
    *   `GuiExplosion`: 物理特效（地震、炮擊）。
    *   `GlobalNavbar`: 顯示連線狀態、氣候預警、族群緊張度。
*   **風格**：沉浸式古風（宣紙底紋、墨跡邊框）。
